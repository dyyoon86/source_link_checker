# Source Link Checker

원천사 ID를 선택하면 관리 대상 서버 여러 대에 **SSH(비밀번호)** 로 접속해서,
그 원천사의 **IP:port 로 유지되는 async 세션이 실제로 연결(ESTABLISHED)되어 있는지**를
`ss`/`netstat` 으로 확인해 **웹 화면**으로 보여주는 도구입니다.

- **순수 Java 1.8 단일 파일** (`src/SourceLinkChecker.java`)
- 웹서버는 JDK 내장 `com.sun.net.httpserver` → **웹 프레임워크 의존성 없음**
- 서버 접속: JSch (`lib/`), Oracle JDBC 는 **DB 조회를 쓸 때만** 필요
- **원천사 목록은 DB 없이도 주입 가능** — FEP 처럼 DB 접속이 막힌 서버를 위해 **외부 JSON 파일**
  또는 **config 직접 내장**으로 원천사 목록을 넣을 수 있습니다. (아래 [원천사 출처](#원천사-출처-파일--config--db) 참고)
- **이중화(다중 세션)** 지원 — 한 원천사가 서로 다른 IP:port 로 2개 이상의 송신 세션을 가지면
  `endpoints` 배열로 각각 점검합니다.
- **서버 상시 주기 점검 + 알람 파일** — 브라우저 없이도 서버가 주기적으로 점검하고,
  상태변화(정상/주의/장애) 시 지정 디렉토리에 알람 로그를 남깁니다.

## 동작 원리

```
[브라우저]
   │  1) GET /api/sources
   ▼
[SourceLinkChecker] ──▶ 원천사 목록(id,name,endpoints) 로드
   │                     출처: sourcesFile(파일) | sources(config) | oracle(DB)  ← 캐시
   │  2) 원천사 선택 후 GET /api/check?id=XXX   (캐시에서 endpoints 사용)
   ▼
   ├─SSH─▶ 서버1 : ( ss -tan ; netstat -an ) | grep ESTAB   → Local/Peer 파싱
   ├─SSH─▶ 서버2 : (동일, 병렬 실행)
   └─SSH─▶ 서버3 : (동일)
   │
   ▼  서버별 × 엔드포인트별 [송신 연결/미연결] + [수신 연결/미연결] + 매칭 소켓 라인 → 화면 카드

[스케줄러]  monitor.enabled=true 면 intervalSec 마다 위 SSH 점검을 자동 수행
   └─ 상태변화 시 → alarmDir/alarm-YYYYMMDD.log 에 한 줄 기록 (브라우저 없이도)
```

- **송신 세션**: 내 서버가 client (원천사로 outbound). Peer(원격) 주소가
  `sendip:sendport` 인 ESTABLISHED 소켓을 찾음.
- **수신 대기 (LISTEN)**: 내 서버가 `recvport` 로 접속 대기 중인지 (Local 포트 = recvport, LISTEN 상태).
- **수신 세션**: 원천사가 내 서버의 `recvport` 로 접속해 들어온 ESTABLISHED 세션 수 (Local 포트 = recvport, 원격 IP 무관).
- `ss -tan` + `netstat -an` 의 ESTABLISHED/LISTEN 라인을 받아 **상태 + Local/Peer 필드로 파싱**해
  판정하므로, 포트 substring 오탐(`:7000` vs `:17000`) 없이 정확합니다. `ss`/`netstat` 어느 쪽이든 동작.

### 단건 조회 / 전체 조회

화면 상단 토글로 두 모드:

- **단건 조회**: 원천사 하나 선택 → 서버별 송신/수신 카드 상세.
- **전체 조회**: 원천사 목록을 **테이블(체크박스)** 로 나열 → 체크한 것들 `선택 연결 조회` →
  **서버별 매트릭스**로 표시. 행=원천사, 열=서버, 각 셀에 `송`(송신)·`수·N`(수신 대기+세션수)
  배지(초록=연결/빨강=미연결/⚠=서버 점검 실패). 행 앞 점으로 정상/주의/장애 종합.
  - **효율**: 서버당 `ss` 를 **1회만** 실행하고 그 출력으로 선택된 모든 원천사를 판정.
    원천사가 많아도 SSH 실행은 서버 수만큼(예: 3대면 3회).

  - **행 클릭 → 서버별 소켓 상세**: 전체 조회 표에서 원천사 행을 클릭하면 그 자리에서
    서버별 상세(송신/수신 카드 + 실제 ss 소켓 라인)가 펼쳐짐(`/api/check` 재사용).

- **자동 주기 점검 + 상태변화 알람**: 전체 조회에서 `자동 점검` 체크 → 주기(10/30/60초/5분)마다
  자동 조회. 원천사 종합상태가 **바뀌면**(예: 정상→장애) 우상단 토스트 알람 + 행 플래시 + 조회 이력에
  '알람'으로 기록. 자동 점검의 주기 조회 자체는 이력에 안 남고 **변화 이벤트만** 남김.
  (기동 시 자동 실행 안 함 — 사용자가 체크해야 시작, 탭/업무 이동 시 자동 중지)

- **조회 이력**: 단건/전체 조회 + 알람 자동 기록. `시각·업무·구분(단건/전체/알람)·대상·결과요약·상태`.
  `<config>.history.json` 으로 영속화(최근 200건), 재시작해도 유지.

### SSH 세션 재사용 (매번 재접속 안 함)

`연결 조회`마다 SSH 핸드셰이크+인증을 새로 하면 느리므로, **세션을 풀에 넣어 살려두고 재사용**합니다.

- **접속은 버튼으로만** — 기동 시 자동 접속하지 않습니다.
  - 하단 **`SSH 세션 → 세션 접속`** 버튼: 현재 업무 서버들에 접속(풀 저장). 서버별 `연결/실패` 표시.
  - `연결 조회`도 세션이 없으면 그때 접속(버튼 동작). 이미 접속돼 있으면 **채널만 열어 빠르게** 실행.
- 접속된 세션은 **keepalive(30초)** 로 유휴에도 유지 → 다음 조회부터 빠름.
- 세션이 끊겼으면 다음 사용 때 **자동 1회 재접속** 후 재시도.
- `끊기` 버튼으로 세션 종료. 프로그램 종료 시에도 전부 정리.

### 원천사 정보 캐싱 (출처 접속 최소화)

원천사 접속정보(IP:port)는 자주 안 바뀌므로 **출처(파일/config/DB)는 처음/재갱신 때만** 읽습니다.

- **캐시 없음 / 첫 조회** → 출처에서 읽어 캐시 저장.
- **캐시 있음** → 출처 접근 없이 캐시 반환 (`최근 갱신 … · 캐시` 표시).
- **↻ 재갱신 버튼** → 출처 재조회 후 캐시 갱신 (`파일/설정/DB에서 갱신됨` 표시).
  - **파일 주입**은 재갱신 때 파일을 다시 읽으므로, 파일만 수정하고 재갱신하면 **재시작 없이** 반영됩니다.
  - **config 내장**도 재갱신 때 config 파일을 다시 읽어 최신 편집을 반영합니다.
- **연결 조회(SSH 체크)는 항상 캐시의 IP:port 사용** — 출처를 건드리지 않습니다.
- 캐시는 config 옆에 JSON 으로 영속화됩니다: `config-prod.json` → **`config-prod.cache.json`**
  (재시작해도 유지, 업무별로 저장). 이 파일은 자동 생성물이라 지워도 다음 조회 때 다시 만들어집니다.

## 원천사 출처 (파일 / config / DB)

업무(`businesses[]`)마다 원천사 목록을 아래 **3가지 중 하나**로 지정합니다. 우선순위는 위에서부터입니다.

| 우선 | 키 | 방식 | 언제 |
|------|-----|------|------|
| 1 | `sourcesFile` | **외부 JSON 파일 주입** | **FEP 등 DB 접속 불가 서버.** 파일만 고치고 재갱신 → 재시작 불필요 |
| 2 | `sources` | **config 에 직접 내장** | 원천사가 몇 개 안 되거나 config 로 관리하고 싶을 때 |
| 3 | `oracle.sourcesSql` | **Oracle DB 조회** | DB 접속이 되는 환경(기존 방식) |

> DB 가 안 되는 FEP 서버라면 `sourcesFile` (또는 `sources`) 을 쓰세요. `ojdbc` jar 도 필요 없습니다.

**`sourceMode` 로 방식 고정(오버라이드):** 업무에 `"sourceMode": "file"|"config"|"db"` 를 넣으면 위 자동
우선순위 대신 그 방식을 강제합니다. 덕분에 `oracle` DB 정보를 **지워지 않고 저장해 둔 채로** `sourceMode:"file"`
로 파일을 쓰다가, 필요할 때 `sourceMode:"db"` 로만 바꿔 DB 조회로 전환할 수 있습니다. (설정 탭의 "원천사 조회 방식"
버튼이 이 값을 설정합니다.)

### 원천사 JSON 형식 (파일 / config 공통) — 송신/수신 분리

파일은 배열 `[ … ]` 또는 `{ "sources": [ … ] }` 둘 다 됩니다. 원천사는 **송신 세션 목록(`sends`)** 과
**수신 포트 목록(`recvs`)** 을 따로 가집니다:

```json
{
  "id": "002",
  "name": "국민은행(송신 이중화, 수신 1포트)",
  "sends": [
    { "label": "주센터",   "sendip": "211.10.0.2", "sendport": "7002", "checkMode": "session", "expectSend": 1 },
    { "label": "재해센터", "sendip": "222.20.0.2", "sendport": "7002", "checkMode": "session", "expectSend": 1 }
  ],
  "recvs": [
    { "label": "수신", "recvport": "9002", "expectRecv": 2 }
  ]
}
```

> **송신/수신 분리**: 송신은 세션마다(이중화면 여러 개), 수신은 포트마다 나열합니다. 여러 송신이 **같은 수신 포트**를
> 써도 수신은 그 포트를 **한 번만** 점검합니다(중복 카운트/설정 혼란 없음). 레거시 `endpoints`(송+수 결합)/flat 형식도
> 자동으로 송·수신으로 분리해 읽습니다.

- **송신 점검 방식** `checkMode` (엔드포인트별): `session`(기본, ESTABLISHED 세션 확인 — **연결유지형**) /
  `skip`(송신 판정 제외, 수신 LISTEN 만 확인). 수신(LISTEN)은 방식과 무관하게 항상 확인합니다.
  - `ss`/`netstat` 로 소켓 상태를 **읽기만** 하며, **원천사 포트로 접속을 시도하지 않습니다**(능동 접속 probe 기능은 제거됨).
  - **DB 방식**은 `sourcesSql` 에 `mode` 컬럼(별칭)을 반환하면 자동 매핑: **A(async)=session**, **S(sync)=skip**.
- **예상 세션 개수** `expectSend` / `expectRecv` (엔드포인트별, 0=미지정): "정상"의 기준 개수. DB 가 없으니 파일/설정에
  저장하고 실제 세션수와 비교합니다. 송신 `expectSend=N` 이면 세션이 **N개 이상**이어야 연결됨(부족하면 `부족 (X/N)`),
  수신 `expectRecv=N` 이면 LISTEN + **N개 이상** 접속돼야 정상(부족하면 `세션 부족`). DB 방식은 `expectsend`/`expectrecv`
  컬럼(별칭)으로 받을 수 있습니다.
- **단일 세션**이면 `endpoints` 없이 `sendip`/`sendport`/`recvport` 만 써도 됩니다(하위호환).
- **이중화(다중 세션)**면 `endpoints` 배열에 세션마다 `{label, sendip, sendport, recvport}` 를 넣습니다.
  - `endpoints` 가 `recvport` 를 생략하면 원천사 최상위 `recvport` 를 공통값으로 상속합니다.
  - `label` 은 화면 표시용(예: 주센터/재해센터, 1번회선/2번회선).
- **DB 방식**도 이중화 지원: `sourcesSql` 이 **같은 id 를 여러 행** 반환하면 그 행들이 한 원천사의
  여러 엔드포인트로 자동 합쳐집니다. (컬럼 alias: `id,name,sendip,sendport,recvport`, 선택 `label`)

### 외부 파일 지정 예시

```json
{ "id": "firm", "name": "펌뱅킹",
  "sourcesFile": "sources-prod-firm.json",
  "servers": [ … ] }
```

`sourcesFile` 은 config 폴더 기준 **상대경로** 또는 **절대경로**. 샘플: `sources-firm.json`,
`sources-prod-firm.json`, `sources-prod-va.json`.

### 설정 탭 (JSON 직접 편집 없이 웹에서 설정)

화면 상단 **`설정`** 탭에서 JSON 을 손대지 않고 폼으로 편집·저장할 수 있습니다.

- **원천사 조회 방식**: `파일 주입 / 설정 내장 / DB 조회` 를 버튼으로 전환. `sourceMode` 필드로 고정되어
  **자동 우선순위를 오버라이드**합니다 — 즉 **DB 접속정보(oracle)를 저장해 둔 채로도 지금은 파일/설정을 쓸 수 있습니다**
  (방식만 바뀌고 데이터는 유지). `sourceMode` 없으면 기존 우선순위(파일 > 설정 > DB) 자동 적용.
- **DB 접속 정보(Oracle)**: JDBC URL·user·password·sourcesSql 편집 → config 에 저장. 지금 방식이 파일/설정이어도
  저장해 두면 나중에 방식만 `DB 조회`로 바꿔 바로 씁니다.
- **DB 조회 → 파일로 내보내기**: 로컬(DB 접속 가능한 곳)에서 DB 를 조회한 결과를 원천사 파일로 저장하고 파일 방식으로
  전환합니다. 그 다음 **DB 에 없는 원천사를 원천사 목록에서 직접 추가·저장**하면 이 파일에 반영됩니다.
  (만든 파일을 FEP 서버로 복사해 그대로 주입 가능)
- **원천사 목록**: 원천사 추가/수정/삭제, `+ 엔드포인트` 로 **이중화 세션** 추가. 저장 위치는 그 업무의
  방식을 따름 — 파일 방식이면 `sourcesFile` 에, 설정 내장이면 config 에 기록(**DB 방식은 편집 불가, 읽기전용**).
- **점검 대상 서버**: SSH 서버(id/host/sshPort/user/password) 추가/수정/삭제 → config 에 저장.
- **주기 점검(monitor)**: 사용 여부·주기·알람 디렉토리·대상 업무 → config 에 저장 후 **스케줄러 즉시 재기동**.
- config 저장 시 `_comment` 등 주석 필드와 포트/주기 정수는 그대로 보존됩니다.
- 저장 즉시 캐시·스케줄러에 반영되어 재시작이 필요 없습니다.

> API: `GET /api/config`(편집용 payload) · `POST /api/config`(`section=sources|servers|monitor|origin|oracle` 저장).

## 동시 조회 폭주 방지 — 모니터 화면 + 스로틀

여러 사용자가 동시에 `선택 연결 조회`를 누르면 서버가 그만큼 SSH 점검을 동시에 돌려 부하가 됩니다. 두 겹으로 막습니다.

### 1) 모니터 화면 (읽기 전용 미러) — `/monitor.html` 또는 `/?mode=monitor`

메인 화면 우상단 `📺 모니터 화면` 링크. **관리자 전체조회 표와 100% 동일한 화면**에서 `조회 이력·설정 탭·조회 버튼·체크박스·행클릭 상세`만 제거한 읽기 전용입니다.

- **미러 전용**: 브라우저는 서버가 주기 점검한 결과(캐시)만 읽습니다(`/api/monitor/latest`). **직접 조회(SSH/DB)를 절대 하지 않습니다.**
- 그래서 **몇 명이 이 화면을 봐도 실제 점검 부하는 `서버 주기당 1회`로 고정**(뷰어 수 무관).
- 업무 탭(펌/가상)으로 하나씩 봅니다. 색/셀/합계/재접속 배지 모두 관리자 화면과 동일. 자동 새로고침(15초).
- `monitor.enabled=true` 가 전제. 꺼져 있으면 안내만 표시(직접 조회로 대체하지 않음).
- API: `GET /api/monitor/latest?biz=<id>` → `{enabled,updatedAt,intervalSec,servers,results}` (부작용 없음).

### 2) 조회 스로틀 (초당 1건) — 관리자 화면에도 적용

관리자가 여러 곳에서 동시에 조회를 눌러도 **실제 SSH 점검은 초당 1건을 넘지 않습니다.**

- `/api/checkall` 은 같은 조회(업무+선택+auto)에 대해 **`minCheckIntervalMs`(기본 1000ms) 내 결과를 재사용**(SSH 안 함).
- 동시 요청은 **single-flight**로 직렬화 → 첫 요청만 실제 점검, 나머지는 그 결과를 공유.
- 즉 50명이 동시에 눌러도 실제 SSH 스윕은 **1회**. (`config.web.minCheckIntervalMs` 로 간격 조절)

> 팁: 상시 감시는 **monitor 를 켜고 모니터 화면을 띄워두는** 조합을 권장 — 점검은 서버가 주기당 한 번, 표시는 모두가 공유.

## 서버 상시 주기 점검 + 알람 파일

브라우저 자동 점검(전체 조회 탭)은 **탭이 열려 있을 때만** 돕니다. FEP 상주 모니터링을 위해
**서버가 스스로 주기 점검**하고 상태변화 시 **알람 로그 파일**을 남기는 스케줄러가 있습니다.

```json
"monitor": {
  "enabled": true,
  "intervalSec": 60,
  "alarmDir": "alarms",
  "businesses": ["firm", "va"]
}
```

- `enabled=true` 면 기동 시 스케줄러 시작 (브라우저 없이도 동작).
- `intervalSec` 마다 대상 업무의 원천사 전체를 SSH 점검(서버당 `ss` 1회).
- 원천사 종합상태가 **바뀌면**(정상↔주의↔장애) `alarmDir` 에 **`alarm-YYYYMMDD.log`** 로 한 줄 append:
  ```
  2026-07-16 14:02:20 | PROD | va/002 | 국민은행(이중화) | 정상 -> 장애 | 송신 0/1 · 수신대기 0/1 · 세션 0
  ```
  (형식: `시각 | 환경 | 업무/원천사id | 원천사명 | 이전 -> 현재 | 요약`)
- `alarmDir` 은 config 폴더 기준 상대/절대경로 (미지정 시 `alarms/`). 디렉토리는 자동 생성.
- `businesses` 를 비우면 전체 업무 대상. 첫 점검은 기준선만 잡고 알람을 내지 않습니다(변화가 있을 때부터).
- 브라우저 자동 점검으로 감지된 상태변화도 **같은 알람 파일**에 함께 기록됩니다.

### 재접속(flap) 감지 — 임시포트 변화

**세션의 클라이언트(임시)포트**를 조회할 때마다 기억해, **개수가 1→1 로 같아 종합상태가 안 바뀌어도**
포트가 달라졌으면 그 사이 **끊겼다 재접속**한 것으로 보고 잡아냅니다.

- **자동 점검·수동 '선택 연결 조회' 모두**에서 감지합니다(같은 '직전 스냅샷' 공유).
  - 수동 조회: 재접속이 있으면 화면 우상단에 **🔄 재접속 감지 토스트**로 즉시 표시.
  - 자동 점검: 브라우저 없이도 알람 로그 파일에 기록.
  - 어느 쪽이든 조회 이력에 `재접속` 구분으로 남고, monitor 가 켜져 있으면 알람 파일에도 기록.
- 원리: 송신은 FEP(로컬)의 임시포트, 수신은 원천사(피어)의 임시포트가 재접속마다 새 값으로 바뀝니다.
- 직전 조회의 임시포트 집합을 **`{config}.ports.json`(파일) + 메모리**에 보관 → 서버 재시작 후에도 이어집니다.
- flap 알람 한 줄 예:
  ```
  2026-07-20 14:02:20 | PROD | va/002 | 국민은행 | 재접속감지(FLAP) | 재접속 감지: [VA-1] 송신 211.10.0.2:7002 임시포트 52344 → 52999
  ```
- 완전 다운(개수 0) 후 재연결은 **상태변화 알람**이 담당하므로 flap 으로 **중복 알람하지 않습니다**.
- 이중화면 링크별로 따로 판정 → 여러 세션 중 **한쪽만 재접속**해도 그 링크만 잡습니다.
- 재접속 이벤트는 조회 이력에도 `flap` 구분으로 남습니다.

## 준비

### 1. jar 배치 (`lib/`)
`lib/README.txt` 참고. 두 개 필요:
- `jsch-0.1.55.jar` (SSH)
- `ojdbc8.jar` (Oracle JDBC, 11g/12c 연결)

### 2. 구조: 환경 × 업무

- **환경(운영/테스트)** = config 파일 단위 (`config-prod.json` / `config-test.json`), 포트로 분리.
- **업무(펌/가상)** = config 안의 `businesses` 배열. 화면 상단 **탭**으로 전환.

```
config-prod.json (운영, :8096)         config-test.json (테스트, :8095)
 └ businesses                            └ businesses
    ├ 펌뱅킹  → oracle(FIRMDB) + servers    ├ 펌뱅킹  → oracle(FIRMTEST) + servers
    └ 가상계좌 → oracle(VADB) + servers      └ 가상계좌 → oracle(VATEST) + servers
```

### 3. config 채우기 (FEP = 파일 주입 권장)
```json
{
  "env": { "name": "운영", "tag": "PROD", "color": "red" },
  "web": { "port": 8096 },
  "ssh": { "connectTimeoutSec": 6, "execTimeoutSec": 10 },
  "monitor": { "enabled": true, "intervalSec": 60, "alarmDir": "alarms", "businesses": ["firm","va"] },
  "businesses": [
    {
      "id": "firm",
      "name": "펌뱅킹",
      "sourcesFile": "sources-prod-firm.json",
      "servers": [
        { "id": "FIRM-1", "host": "10.7.1.10", "sshPort": 22, "user": "root", "password": "pw1" }
      ]
    },
    {
      "id": "va",
      "name": "가상계좌",
      "sources": [
        { "id": "001", "name": "신한은행", "sendip": "211.10.0.1", "sendport": "7001", "recvport": "9001" },
        { "id": "002", "name": "국민은행(이중화)", "recvport": "9002",
          "endpoints": [
            { "label": "주센터",   "sendip": "211.10.0.2", "sendport": "7002" },
            { "label": "재해센터", "sendip": "222.20.0.2", "sendport": "7002" }
          ] }
      ],
      "servers": [
        { "id": "VA-1", "host": "10.7.2.10", "sshPort": 22, "user": "root", "password": "pw1" }
      ]
    }
  ]
}
```

- `env`: 화면 배너/탭 제목 (운영=red, 테스트=blue). `ssh`: 전역 타임아웃. `monitor`: [서버 상시 점검](#서버-상시-주기-점검--알람-파일).
- `businesses[]`: 업무마다 **원천사 출처**(`sourcesFile` / `sources` / `oracle`) 와 `servers`(대상 서버) 를 가짐.
  업무를 늘리려면 배열에 추가(탭 자동 생성). 1개면 탭 숨김.
- 위 예시: 펌뱅킹=**파일 주입**, 가상계좌=**config 내장(002는 이중화)**. DB 를 쓰려면 [원천사 출처](#원천사-출처-파일--config--db) 참고.

> **송신/수신 판정 기준**
> - **송신(send)**: 내 서버 → 원천사. `sendip:sendport` 를 **Peer(원격) 주소**로 갖는 ESTABLISHED 세션.
> - **수신(recv)**: 원천사 → 내 서버. `recvport` 를 **Local(로컬) 포트**로 갖는 ESTABLISHED 세션 (IP 무관).
> - **이중화**면 엔드포인트별로 각각 판정하고, 서버 종합상태는 **모든 송신/수신 엔드포인트가 정상일 때** 정상입니다.
> - **수신 포트 합산**: 송신은 다르지만 여러 엔드포인트가 **같은 수신 포트**를 쓰면, 그 포트의 수신 세션은
>   합쳐서 **한 번만** 셉니다(엔드포인트마다 중복 카운트하지 않음). 송신은 엔드포인트별로 각각 셉니다.
> - **연결유지 여부(checkMode)**: `ss` 세션 스냅샷은 **연결유지형(session)** 에만 유효합니다. 트랜잭션마다
>   접속/종료하는 **비연결유지** 링크는 `skip`(제외)으로 두면 됩니다.
>   skip 이면 그 축은 정상/장애 판정에서 빠지고, 화면에는 `해당없음`/`제외`로 표시됩니다.

## 환경 분리 (운영 / 테스트)

DB·서버 IP 가 환경마다 다르므로 **환경별 config 파일**로 분리합니다.

| 환경 | config 파일 | 포트 | 실행 | 배너 |
|------|-------------|------|------|------|
| 테스트 | `config-test.json` | 8095 | `run-test.bat` | 파란색 TEST |
| 운영 | `config-prod.json` | 8096 | `run-prod.bat` | 빨간색 PROD ⚠ |

- 각 config 의 `env` 로 화면 상단 배너(운영=빨강/테스트=파랑)와 콘솔·탭 제목이 결정됩니다.
  ```json
  "env": { "name": "운영", "tag": "PROD", "color": "red" }
  ```
- 포트가 달라서 **테스트·운영을 동시에 띄워** 비교할 수 있습니다 (8095 / 8096).
- 실수 방지: 운영 화면은 상단에 빨간 `⚠ 운영 환경` 배너가 항상 표시됩니다.
- `run.bat test` / `run.bat prod` 로도 실행 가능 (인수 없으면 `config.json`).

## 빌드 & 실행

```bat
build.bat            REM  javac -> classes\  (JDK 1.8/11/21 호환)

run-test.bat         REM  테스트 환경  http://localhost:8095
run-prod.bat         REM  운영 환경    http://localhost:8096
```

브라우저에서 해당 포트 접속 → 원천사 선택 → **연결 조회**.
(빌드/실행 중 오류가 나면 창이 닫히지 않고 `[ERROR]`/`[build] FAILED` 메시지가 남습니다.)

## 화면 (Hecto Financial 디자인 시스템 — 라이트 테마)

- **히어로**: 선택한 원천사 + 엔드포인트별 송신 대상 `ip:port` / 수신 포트 `:port` 강조
  (이중화면 `이중화 N세션` 배지와 함께 세션마다 표시).
- **요약 타일**: 송신 연결 N/M · 수신 대기(LISTEN) N/M · 수신 세션 합계 · 점검 실패.
- **서버 카드** (좌측 상태 컬러바: 정상=초록 / 주의=주황 / 실패=빨강):
  - 이중화면 카드 안에 `◆ 주센터` / `◆ 재해센터` 처럼 **세션별로 구분**해 송신/수신 행을 표시.
  - **송신**: `● 연결됨` / `● 미연결` + 대상 `ip:port` 칩
  - **수신**: `● 대기중 (LISTEN)` / `● 닫힘` + `세션 N개` 배지
  - `▸ 소켓 N건`: 접이식(details)으로 매칭된 실제 ss/netstat 라인 표시 (강제 스크롤 없음)
  - `⚠ 점검 실패`: SSH 접속/명령 실행 실패 시 원인 표시

## 참고 / 확장 여지

- **판정 방식이 SSH+ss/netstat** 이므로, 서버에 `ss` 나 `netstat` 둘 중 하나는 있어야 함.
- 구형 서버라 JSch 기본 KEX/암호로 SSH 핸드셰이크가 안 되면, `lib/` 의 JSch 를
  mwiede 포크 최신 버전으로 교체하면 대부분 해결됨.
- 방향을 반대(원천사 → 내 서버)로 봐야 하면 `SourceLinkChecker.java` 의 `epRx`/`buildCmd`
  부분에서 필터를 바꾸면 됨 (현재는 라인 전체 grep 이라 양방향 모두 잡힘).
- **알람 파일 포맷**은 `writeAlarmFile()` 에서 한 줄 포맷(`시각 | 환경 | 업무/id | 이름 | 이전 -> 현재 | 요약`)을
  바꾸면 됨. 사내 알림 연동(웹훅/메일)이 필요하면 이 지점에서 확장하면 됩니다.
